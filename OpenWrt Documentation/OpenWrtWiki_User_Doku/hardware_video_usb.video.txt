====== USB Video Support ======
The word "video" can refer to
  - a video card aka graphics card = graphic adapter
  - a video accelerator card = DE-/CODEC accelerator
  - a **video camera**. This page is all about OpenWrt support for **web cams** and **similar devices** (i.e. something with a video stream, e.g. a **TV card**)!

===== Preparation =====
Please inform yourself upstream:
  - [[http://linuxtv.org/wiki/index.php/Webcam_Devices]] teaches us, that there are four types of web cams:
    - those that have //UVC support//  ->[[http://www.ideasonboard.org/uvc/|Linux UVC driver and tools]]
    - those that have //GSPCA support// ->[[http://linuxtv.org/wiki/index.php/Gspca]]
    - those that have //out-of-tree support//
    - those that have //no Linux support//
  - Then there is the framework [[wp>Video4Linux]] and a whole load of applications that work with V4L version 1 and/or version 2.
    - [[http://lwn.net/Articles/203924/|The Video4Linux2 API: an introduction]]
    - [[http://linuxtv.org/wiki/index.php/Main_Page]]
    - [[http://v4l2spec.bytesex.org/spec/|V4L2 API Specification]]

==== Prerequisites ====
  - Follow [[docs:guide-user:storage:usb-installing]] to obtain basic USB support
  - Plug the USB webcam of your choice into the USB port of your OpenWrt router

===== Installation =====
Install ''kmod-video-uvc'' or ''kmod-video-gspca-core'' + matching driver via [[docs:guide-user:additional-software:opkg]].

===== Configuration =====
There's nothing to configure.

===== Applications =====
Application can work directly with the driver, or with V4L1/V4L2.
